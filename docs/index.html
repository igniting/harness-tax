The Harness Tax

Wrapping a model in an agentic harness multiplies its context 40–125× and its per-run cost 5–8×. Sometimes that buys a 28-point jump in pass@1 — and sometimes it buys nothing. A micro-benchmark of raw API calls vs Claude Code on five contamination-resistant coding problems.

The question

Almost nobody calls a frontier model raw anymore. Production coding assistants wrap the model in a harness: a long system prompt, a catalog of tool schemas, a turn loop, permission machinery, prompt-cache churn. That wrapper is invisible in benchmark tables, which report the model, not the model-plus-harness.

So we measured it directly. Same five problems, same models, same effort setting, three configurations:

  • Raw API — one messages.create call, extract the code block, grade it.
  • Harness, single-shot — Claude Code headless (claude -p), tools restricted to Write: write solution.py and stop. Isolates the fixed cost of the harness: system prompt, tool schemas, turn loop, cache accounting.
  • Harness, agentic — full local toolset: the model can write scratch tests, run them, and iterate until it is confident. Isolates what iterate-and-self-verify buys, and costs.

The delta between the first two is the harness tax (tokens, dollars) and the harness lift (pass rate). The delta to the third is the price of self-verification.

Method

Contamination-resistant problems, generated with Fable 5

Public benchmark problems leak into training data, and a memorized solution tells you nothing about the harness. So the five problems here were generated with Claude Fable 5 as invented specifications — deliberately novel semantics that don't exist in any codebase to memorize: a made-up expression language ("Calc-26") with a midpoint operator and non-standard unary-minus precedence, a rate limiter with weighted refund-on-reject semantics, a recurrence expander pinned to real 2026 DST transitions, a fully-determined greedy DAG scheduler, and a find-the-five-planted-bugs cache module.

ProblemTests forOracle
p1_rate_limiterstateful invariants, boundary semanticsdifferential vs brute-force reference on random streams
p2_expr_parserprecedence with a novel operator, adversarial spec quirkrandom-AST differential + pinned spec examples
p3_recurrenceDST gaps/overlaps, half-hour zonesindependent minute-scan around real 2026 transitions
p4_schedulerdeterministic multi-worker DAG simulation, tie-breakingindependent unit-time-step simulator
p5_debug_cachefind-and-fix 5 planted bugs, API preservedindependent list-based oracle + targeted units

Grading is by automated oracle, not string match — mostly differential testing against an independent reference implementation on randomized inputs. For p1 and p2 the repo ships mutants (plausibly-wrong solutions), and harness/validate.py proves the oracles kill them. If the oracle can't catch a subtly wrong solution, the pass rate is noise; this check makes the pass rates mean something.

Setup

  • Models: Opus 4-6, 4-7, 4-8 ($5/$25 per MTok) and Sonnet 5 ($2/$10 per MTok), all at effort=high with adaptive thinking; sampling params left at defaults.
  • Raw API: 5 samples per model × problem (100 runs). Output tokens include thinking.
  • Harness single-shot: Claude Code 2.1.201 headless, --allowedTools Write, subagents disabled, 3 samples per cell (45 runs).
  • Harness agentic: full toolset in an isolated temp workspace, subagents disabled, 1 sample per cell (15 runs).
  • Context accounting: harness context = input + cache-creation + cache-read tokens summed over all turns, as reported by Claude Code; cost is its reported total_cost_usd. The comparison ratio is harness context ÷ raw-API input tokens.

All numbers below come from the JSONL files in results/; every transcript is in the repo.

Results

Baseline: the raw API is cheap, and newer isn't better

One API call per problem, code block extracted and graded. Two things stand out. First, within the Opus line, raw single-shot pass@1 fell across releases on these novel specs: 92% (4-6) → 80% (4-7) → 72% (4-8). Second, Sonnet 5 — the cheapest model tested — went 25 for 25, spending ~4× the output tokens (it thinks longer) but still costing about the same per run.

The failures cluster on p2 (the novel-precedence parser) and p3 (DST edge cases) — exactly the problems where an invented spec contradicts what training data suggests. More on that below.

The lift: what the harness buys

Now the same problems through Claude Code. Opus 4-7 was dropped from harness runs after its raw baseline; the three remaining models each ran raw, single-shot, and agentic.

The pattern: the harness rescues models that struggle raw, and does nothing for models that don't. Opus 4-8 jumps 28 points to 100% from the single-shot harness alone — no iteration, no test-running, just the harness's system prompt and file-writing frame around the same model. Its raw failures on p2 (40%) and p3 (20%) vanish entirely. Opus 4-6 moves one point. Sonnet 5, already perfect, has nowhere to go.

And in agentic mode — where the model can write scratch tests, run them, and fix what fails — everything converges to 100%.

The tax: what the harness costs

A raw call on these problems sends 400–1,000 input tokens. The same problem through the single-shot harness touches 36,000–71,000 context tokens per run — the system prompt, the tool catalog, and every turn's cache writes and reads, again and again across the loop.

The floor — roughly 36–46K tokens even on the smallest problems — is the fixed furniture: system prompt plus tool schemas, re-read from cache on every turn. Variation above the floor is extra turns compounding the cache reads.

The 1,037× point is real and worth dwelling on. Two of Sonnet 5's three single-shot runs on p3 ignored "write the file and stop," ran 13 turns, and accumulated up to 1,016,995 context tokens — a $2.94 run for a problem the same model solves raw for five cents. Both runs still passed. This is the harness tax at its worst: the loop gives the model room to spiral, and the cache accounting bills every lap.

Cost: per run vs per solve

Per run, the harness costs 5–8× the raw call. But per solved problem, the picture inverts for the model the harness helps: Opus 4-8 raw wastes 28% of its calls on failures ($0.042 per solve at 72%), while through the harness every $0.15 run lands. If a failure means a human retries or debugs, the harness is the cheaper path for that model — and pure overhead for Sonnet 5, which was already at 100% for a nickel.

Sonnet 5's single-shot average ($0.39) carries the two runaway p3 runs; its median single-shot run cost ~$0.16. In agentic mode, where it can satisfy itself by actually running a test instead of re-deriving everything in-context, it was the cheapest model at $0.198 per problem.

Agentic mode: reliability, not capability

Agentic mode is where the harness earns its keep differently: not by making the model smarter, but by letting it check. Every model reached 100%, taking 2–10 turns per problem.

Opus 4-6's 10-turn slog on p4 (the scheduler whose tie-breaking must match the spec exactly) is the shape of agentic self-repair: write, test, find the tie-break wrong, fix, repeat. It costs $0.44 — 14× the raw call — and it works. The right mental model is insurance: 2–10× the single-shot price for a correctness guarantee, worth it when a wrong answer is expensive, wasteful as a default.

Findings

1. The tax is real, but it's not just overhead

40–125× context sounds damning, and for a model that was already passing, it is — pure cost, zero lift. But for Opus 4-8 the same overhead bought +28 points of pass@1. Whether the harness is worth it is a property of the model × task, not of the harness. Measure yours.

2. The harness lift comes from the frame, not the tools

Opus 4-8's jump to 100% happened in single-shot mode — the model couldn't run anything. The system prompt and the write-a-file framing alone changed how it read the spec. That's a cheap, underrated knob: before reaching for full agentic loops, try the frame.

3. Cheaper models can beat expensive ones — pick by measured pass rate, not price tier

Sonnet 5, at 40% of Opus's token price, outscored every Opus release on raw calls (100% vs 72–92%) and was the cheapest route to 100% in agentic mode. It spends ~4× the output tokens thinking, and the price gap swallows that whole.

4. On novel specs, models default to their training priors

The p2 spec — invented by Fable 5 precisely to have no precedent — says unary minus binds tighter than exponentiation:

-2 ^ 2 = 4  (the spec)   vs   -2 ^ 2 = -4  (Python, Ruby, math convention)

Opus 4-8 read the spec, then implemented the convention anyway — in 3 of 5 raw runs. The oracle's pinned spec examples caught it every time. This is the cleanest failure signature in the whole benchmark: when a specification contradicts a strong prior, raw single-shot generation follows the prior. Both the harness frame and agentic self-testing fixed it — which suggests the fix isn't more capability, it's anything that forces a second read.

Caveats

  • Small n. 5 problems, 3–5 samples per cell (1 for agentic). Deltas under ~10 points are within noise; treat this as a signal and a method, not a leaderboard.
  • One harness, one version. Claude Code 2.1.201, subagents disabled. Other harnesses (and other versions) will have different fixed costs and different frames.
  • Cost accounting differs by lane. Raw cost is computed from list prices; harness cost is Claude Code's self-reported total, which includes cache-write premiums and cache-read discounts.
  • The problems are narrow by design. Contamination resistance means small, spec-dense, self-contained tasks — the regime where a raw call is even viable. On large-codebase work the raw-API lane doesn't exist, and the harness tax is simply the price of entry.

Reproduce it

git clone https://github.com/igniting/harness-tax && cd harness-tax
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-...

# prove the oracles kill the mutants (no API needed)
python3 harness/validate.py

# raw API baseline
python3 harness/runner.py --models claude-opus-4-6 claude-opus-4-8 claude-sonnet-5 \
    --samples 5 --effort high --out results/run1.jsonl

# harness, single-shot and agentic
npm install -g @anthropic-ai/claude-code
python3 harness/cc_runner.py --models claude-opus-4-6 claude-opus-4-8 claude-sonnet-5 \
    --samples 3 --subagents deny
python3 harness/cc_runner.py --models claude-opus-4-6 claude-opus-4-8 claude-sonnet-5 \
    --samples 1 --agentic --subagents deny

# compare
python3 harness/cc_report.py results/cc_results.jsonl results/run1.jsonl
